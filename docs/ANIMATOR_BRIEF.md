# SAGO AI — 애니메이터 작업 브리프

> 이 문서 하나로 프로젝트를 파악하고 작업에 바로 들어갈 수 있도록 작성했습니다.

---

## 1. 프로젝트가 뭔가요?

**SAGO AI**는 아성다이소 매장의 산업재해를 사전에 막기 위한 AI 알림 시스템입니다.

매일 아침, AI가 각 매장의 날씨·물동량·과거 사고 이력을 분석해  
**"오늘 이 매장에서 낙상 위험이 높다"** 같은 판단을 내리고,  
매장 관리자·직원에게 카카오톡으로 예방 알림을 보냅니다.

---

## 2. 캐릭터가 어디에 쓰이나요?

알림은 2단계로 전달됩니다.

```
① 카카오톡 피드 카드 (정지컷)
   └─ 캐릭터 일러스트 + 사고 유형 + 안전 수칙 요약

      [안전가이드 전체 보기 ›] 탭

② 모바일 웹 랜딩 페이지 (움직이는 캐릭터)
   └─ 캐릭터가 해당 사고 시나리오를 실제로 연기
      아래 스크롤 → 오늘의 안전 수칙 상세
```

캐릭터는 **알림의 얼굴**입니다.  
수신자가 카드를 탭하면 제일 먼저 보이는 것이 이 캐릭터의 모션입니다.  
딱딱한 안전 공문이 아니라 "우리 직원"이 직접 보여주는 느낌이 목표입니다.

## 3. 캐릭터 디자인 스펙

| 항목 | 내용 |
|------|------|
| 성별·직종 | 여성, 다이소 매장 직원 |
| 복장 | 빨간 셔츠 + 파란 앞치마 + 명찰 |
| 헤어 | 검은 단발 포니테일 |
| 스타일 | 플랫 일러스트 (카툰/벡터). 귀엽되 가볍지 않게 — 안전이 주제이므로 |
| 표정 | 5종 — 기본·충격·통증·경고·안전(밝음) |
| 뷰포트 | 세로형, 캐릭터 전신이 들어오는 비율 |

## 4. 필요한 애니메이션 — 10개 시나리오

각 시나리오는 **3단계 스토리 구조**로 재생됩니다.

```
상황 등장  →  캐릭터 반응  →  충격·결과
(이펙트)      (몸 움직임)     (이펙트 + 굳음)
```

루프 재생 (랜딩 페이지에서 계속 반복).

| # | slug | 이름 | 표정 | 핵심 모션 | 이펙트 |
|---|------|------|------|-----------|--------|
| 1 | `fall` | 낙상 | 충격 | 균형 잃음 → 뒤로 쓰러짐 → 착지 충격 | 바닥 충격 별·먼지 |
| 2 | `slip` | 미끄러짐 | 충격 | 발 미끄러짐 → 다리 벌어지며 뒤로 넘어짐 | 물 웅덩이 튐·미끄럼 주의 마크 |
| 3 | `collision` | 충돌 | 충격 | 걷다가 충돌 → 뒤로 튕겨남 · 휘청 | 충돌 스파크 |
| 4 | `property` | 재물손상 | 충격 | 머리 위에서 상자 낙하 → 머리 보호 자세 | 상자 낙하 → 충격파 |
| 5 | `cut` | 자상 | 통증 | 작업 중 → 베임 순간 팔 급반동 → 손 감싸쥠 | 빨간 사선 + 통증 별 |
| 6 | `strain` | 무리한동작 | 통증 | 상자 들기(앞으로 굽힘) → 허리 통증(뒤로 젖혀짐) | 허리 번개 ⚡ |
| 7 | `health` | 건강장해 | 통증 | 가슴에 손 → 무릎 꺾이며 주저앉음 | 심장 펄스 + ECG선 |
| 8 | `claim` | 고객불만 | 경고 | 뒤로 물러서며 양팔 달래는 제스처 | 말풍선 ⚠ 흔들림 |
| 9 | `default` | 경고 | 경고 | 팔 들어 경고 포인팅 | 경고 삼각형 펄스 |
| 10 | `safe` | 안전 | 밝음 | 신나게 손 흔들기 + 살짝 hop | 초록 체크 팝인 + 반짝임 |

## 5. 납품 형식

### 필수
- **Rive 파일 1개** (`daiso_worker.riv`)
- 파일 안에 **State Machine 1개** — 입력: `String` 타입 `scenario` (값: `fall` / `slip` / `collision` / `property` / `cut` / `strain` / `health` / `claim` / `default` / `safe`)
- 각 시나리오 → 해당 애니메이션 클립으로 자동 전환

### 권장 납품 구조
```
daiso_worker.riv
  └─ Artboard: "Worker"
       ├─ State Machine: "ScenarioMachine"
       │    └─ Input: scenario (String)
       │         ├─ "idle"    → idle_breathing 클립
       │         ├─ "fall"    → fall 클립
       │         ├─ "slip"    → slip 클립
       │         └─ ... (10개)
       └─ Animations (클립 11개: idle + 10 시나리오)
```

### 선택 (있으면 좋음)
- 각 시나리오 PNG 정지컷 1장 (카카오톡 카드용 — 가장 임팩트 있는 프레임)

## 6. 기술 연동 방법 (개발자 참고)

납품받은 `.riv` 파일은 다음 컴포넌트에 바로 꽂힙니다.

```jsx
// proj/src/components/character/CharacterPlayer.jsx
// 우선순위: Rive(.riv) → CSS SVG(현재 프로토타입) → PNG(폴백)
<CharacterPlayer scenario="fall" />
```

Rive 런타임은 `@rive-app/react-canvas` 사용 예정.  
파일 위치: `proj/public/character/daiso_worker.riv`
